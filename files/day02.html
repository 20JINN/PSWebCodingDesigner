실제 웹 개발에서는 span의 display나 position 등 다양한 속성 뿐만아니라 그 부모 엘리먼트의 속성에 따라서도 결과물이 달라질 수 있기에 이게 정답이라고 단언할 수는 없지만 저는 위 예시 그림을 만들며 발생한 문제를 이렇게 //span태그를 공백없이 붙여 쓰거나, float:left;를 사용하여// 해결했다는 경험을 공유하기 위해 만들었습니당.
☀ 1.span with whitespace
@html
<span id="r1"></span>
<span id="r2"></span>
<span id="r3"></span>

@CSS
#r1, #r3{
width: 30px;
height: 30px;
display: inline-block;
background-color: rgba(62, 24, 199, 0.28);
}
#r2 {
width: 228px;
height: 30px;
display: inline-block;
background-color: rgba(107, 115, 126, 0.3);
}

#z-index 예시
☀ z-index
The z-index CSS property specifies the z-order of a positioned element and its descendants.
When elements overlap, z-order determines which one covers the other. An element with a larger z-index generally covers an element with a lower one.

z-index는 z축의 order을 나타낸다.
요소들이 overlap될 때, z-order는 다른 무언가를 덮는 어떤 것을 결정한다.
z-index가 큰 요소는 일반적으로 그것보다 z-index가 낮은 요소보다 상위에 위치한다.

auto 입력 값
z-index: auto;

뜻
The stack level of the generated box in the current stacking context is the same as its parent's box.

정수로 된 수치(integer)
z-index: 0;
z-index: 3;
z-index: 289;
z-index: -1; /* Negative values to lower the priority */

뜻
This integer is the stack level of the generated box in the current stacking context. The box also establishes a local stacking context in which its stack level is 0. This means that the z-indexes of descendants are not compared to the z-indexes of elements outside this element.
정수로 된 수치는 해당 영역에서 stack level을 0부터 만들 수 있다. 이 영역 밖에서 z-index를 가진 요소는 해당 영역 안에서 하위 z-index를 가진 요소에게 영향을 끼치지 않는다.

Global 입력 값
z-index: inherit; 부모의 z-index를 이어 받는다.
z-index: initial; 이 요소의 z-index를 가장 하위의 요소로 둔다.
z-index: unset; 부모로부터 상속된게 있으면, inherit 적용/ 아니면 initial 적용
