시멘틱 마크업의 이해

우리가 배우는 HTML은 Hyper Text Markup Language의 약자입니다.


이 중 Markup은 하나의 언어인데, 태그 등을 이용하여 문서나 데이터의 구조를 명기하는 언어입니다. HTML은 이러한 Markup 언어의 일종으로, 웹문서의 내용과 구조를 특정 태그들로 구성해 정보를 구조화 합니다.


Semantic은 '의미론적인'이라는 뜻을 가지고 있는 단어입니다. 해석하자면 '의미가 분명한 마크업'을 바로 시맨틱 마크업, 혹은 의미론적인 마크업이라고 할 수 있겠습니다.

마크업이라는 행위 자체가 바로 추가적인 정보를 다는 행위인데 그것의 정보가 올바르지 않다면 아무리 시각적으로 훌륭하고 UI가 뛰어난웹 사이트라 하더라도 좋은 품질의 마크업으로 이뤄진 것이라 할 수 없습니다. 따라서 가장 좋은 마크업은 사용자가 쉽게 접근할 수 있는 마크업, 올바른 정보를 갖춘 마크업입니다. 우리가 배운 div 태그는 아무런 의미론적 의미가 존재하지않습니다. 이제 우리는 시멘틱 마크업을 통해 해당 페이지를 의미론적인 페이지로 탈바꿈시켜야 합니다.

현재 여러분이 보는 페이지의 뷰는 이전과 똑같겠지만, 코드를 보시면 더 의미가 분명한 마크업을 보실 수 있을 것입니다.

<Selecting specific tag>
☀ Select with 'Class'

First, define class name to a tag in html.
And then in CSS, write down properties with .Classname.

@html
<div class="클래스이름">Hello world</div>

@CSS
.클래스이름 {
color: red;
background: yellow;
}
☀ Select with 'Id'

Define id to a tag you want to select in html.
Use #Id and give some style tag in CSS.

@html
<div id="아이디이름">유일한이름입니다</div>

@CSS
#아이디이름 { color: blue; }
<The 'Display' property>

display is CSS's most important property for controlling layout. Every element has a default display value depending on what type of element it is.
The default for most elements is usually block or inline.
A block element is often called a block-level element and an inline element is always just called an inline element.

☀ inline : 현재 글줄에 이어서 표현됩니다
-display: inline
-margin-top, margin-bottom이 적용되지 않습니다.
-컨텐츠 고유의 사이즈로 표현됩니다.
☀ block : 현재 글줄 다음에 Block단위로 표현됩니다
-display: block
-부모 요소의 100% width를 차지합니다
-margin이 적용됩니다
☀ inline-block : 글줄에 이어서 표현하되 Block단위로 표현됩니다
-display: inline-block
-컨텐츠 고유의 사이즈로 표현됩니다.
-margin이 적용됩니다.
☀ none : 문서에서 보이지 않게 만듭니다
-display: none
