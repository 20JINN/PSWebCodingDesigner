<API : behance api 연습하기>

API는 Application Programming Interface의 약자로 응용 프로그램에서 사용할 수 있도록, 서비스가 제공하는 기능을 제어할 수 있게 만든 프로그래밍 인터페이스를 뜻한다.

포스트맨 설치 (코드 없이 API테스트를 할 수 있다.)
1. https://www.behance.net/dev 사이트 방문
2. Manage Your Apps > Register a New App 선택
3. Application Name 입력
4. Website는 우선 http://localhost 입력
5. Description 아무거나 입력
6. client id 복사해놓기
7. 그 후 Endpoint 메뉴로 이동
8. 그 중에서 user항목으로 이동
9. /v2/users/user/appreciations 항목의 Example 보기
10. https://api.behance.net/v2/users/{유저ID}/projects?client_id={본인의_클라이언트ID}
11. https://api.behance.net/v2/users/anzi/projects?client_id=SwLM8mfOl2ItSAioRf2DSdzcTFE00wJf

<Build Tool 사용하기 (feat. Parcel.js)>

Parcel.js는 웹사이트 Build Tool입니다. Task Runner라고도 합니다.원래 웹은 Build Tool이 따로 필요가 없었습니다. 그냥 서버에 HTML, CSS, Javascript 파일만 올리면 되었기 때문입니다.
하지만 웹이 발전 하면서 웹에 올리기 전에 (혹은 개발 하는 중에) 필요한 작업들 이 생겨나게 되었고 그래서 최근에는 Build Tool을 많이 사용하고 있습니다.

☀parcel.js 설치

https://nodejs.org/ko/ 1. nodejs 안정적 버전을 다운로드 후 설치
2. terminal을 열고 npm install -g parcel-bundler 를 입력하여 설치합니다.
3. 안 될 경우 명령어 앞에 sudo를 붙입니다. (관리자 권한으로 실행) sudo npm install -g parcel-bundler

Terminal에서 명령어를 치면 빌드가 되고 localhost:1234에서 사이트를 확인 할 수 있습니다.

~$ parcel index.html

진짜 배포를 위한 빌드를 할때 아래 명령어를 사용합니다

parcel build index.html

완료하면 dist 폴더에 배포를 위한 모든 파일이 준비됩니다.


이런 일들을 수월하게 하기 위해서 Build Tool을 사용합니다.
  • 개발할 때 코드 수정 시 자동으로 reload해준다.
  • 속도를 빠르게 하기 위해 파일의 코드들을 최적화한다. (minify, uglify) → 용량 절약
  • javascript가 버전업 되면서(ES6) 최신 버전의 문법을 쓰면 과거 브라우저에는 작동하지 않게 되는데 ES6로 작성된 것을 ES5로 변환 해준다.
    (최신 브라우저에서는 변환 없이도 바로 동작하기도 합니다)
  • javascript 코드양이 기하급수적으로 많아지다 보면 파일을 나누어서 관리해야 하는데 js는 이를 지원하지 않기 때문에 module 기능을 사용하려면 Build Tool이 필요하다.
  • 여러 파일로 쪼개어 작업한 것을 하나의 파일로 합쳐준다.
  • CSS가 불편하니 SASS라는 것을 쓰게 되고 SASS로 작성된 것을 CSS로 변환해준다. (우린 배우지 않습니다)
  • ... 그 외 많은 기능들이 있습니다.


이런 일들이 필요해지게 되었고 이런 일을 자동화 해주는 것이 Build Tool이라고 쉽게 생각해봅시다.
만약 이런 툴이 없었다면 우리는 위에서 나열한 일들을 배포할 때마다 일일이 작업해 주어야 할 것 입니다.


Build Tool에는 여러가지가 있습니다. 아래는 대표적인 3가지입니다.
  • Webpack : 아마 현업에 있는 개발자들이 가장 많이 사용하는 빌드 툴 일 것입니다.
  • 우리가 쓰지 않는 이유는 설정이 너무 어렵고 복잡하기 때문입니다.
    Build Tool 초기 설정은 보통 프로젝트 작업 시 자주 필요한 것은 아니기 때문에, 회사의 프로젝트에 기여하고 싶을 때에는 개발자에게 어떻게 Build하는지 물어보면 알려줄 겁니다.
  • Gulp, Grunt : 이 것들도 많이 씁니다만, Webpack보다 먼저 나와서 대세가 되었고 지금은 Webpack이 대세입니다. (또 금방 바뀔지도 모르죠...)
  • Parcel.js : 오래되지 않았지만 가장 간편하고 대다수의 기능을 지원해줍니다. Webpack에 비해 안정성이 떨어집니다. 중요 프로젝트에는 권장하지 않습니다.
